HOW TO READ THIS DASHBOARD

This dashboard visualizes 5,000 healthcare ransomware incidents across 2024. Three bidirectionally linked views let you explore temporal patterns, attack pathways, and incident-level details.

VIEW 1: ECG MONITOR

Each peak represents one month of 2024. The waveform shape encodes the cybersecurity severity of that month:

Normal Sinus Rhythm

SEVERITY 0-40. Strong, regular P-QRS-T pattern.
= Low infection rates, fast recovery, backups intact.

Arrhythmia

SEVERITY 40-65. Irregular rhythm, variable peaks.
= Rising infections, slower recovery, some backups compromised.

Cardiac Arrest

SEVERITY 65-100. Chaotic fibrillation to flatline.
= Severe infections, weeks of downtime, data unrecoverable.

Severity Score Calculation

Each incident's severity is a weighted composite of four factors:

Infection Rate (30%): ransomware infection rate, normalized against max (90%).

Recovery Time (25%): days to recover, normalized against max (120 days).

Data Loss (25%): inverse of data restored %. Higher loss = higher severity.

Backup Compromise (20%): binary. Compromised backups add full 20%.

Monthly scores are averaged, then stretched to 0-100 using percentile normalization. Months with n=0 incidents flatline.

Interactions

Brush across months to filter both views below. Select one or more organization types to filter all views. Incident counts (n=) update below each month.

VIEW 2: ATTACK PATHOLOGY (SANKEY DIAGRAM)

Traces how incidents flow through six stages from entry to outcome. Band thickness shows proportional flow volume.

Stage 1: Entry Vector

How the ransomware initially gained access to the organization:

Compromised Credentials: attackers obtained valid login credentials (stolen passwords, credential stuffing, purchased from dark web).

Exploited Vulnerability: attackers exploited an unpatched software vulnerability (outdated systems, zero-day exploits).

Phishing Email: malicious email tricked a user into clicking a link or opening an attachment.

RDP Exploit: attackers gained access through exposed Remote Desktop Protocol services.

Malicious Website: drive-by download or watering hole attack through a compromised website.

Infected Removable Media: USB drives or other physical media carrying malware.

Stage 2: Data Encryption

Whether the ransomware successfully encrypted the organization's data. Yes = encrypted. No = not encrypted.

Stage 3: Data Stolen

Whether attackers exfiltrated (stole) copies of the data before or during encryption. This is critical for healthcare as stolen patient records trigger HIPAA violations and privacy breaches.

Stage 4: Backup Lost

Whether the organization's backup systems were also compromised. Backup loss dramatically increases recovery time and ransom payment likelihood.

Stage 5: Paid Ransom

Whether the organization paid the ransom demand. Payment does not guarantee data recovery.

Stage 6: Recovery Outcome

Percentage of data the organization ultimately recovered:

< 25%: catastrophic loss. 25-50%: severe loss. 50-75%: partial recovery. 75-100%: good recovery.

Interactions

Click nodes to build a path filter (multi-select per column, OR within column, AND across columns). Band thickness adjusts to show filtered proportions. Hover over any node to see incident count and percentage. Nodes with no matching incidents show a red warning on hover.

VIEW 3: INCIDENT EXPLORER

Scatter Mode

Each dot is one ransomware incident, colored by severity (green = low, red = critical). Select any two numeric dimensions for X and Y axes: Infection Rate, Recovery Days, Data Restored, Facilities Affected, or Incidents count. Brush to select dots and see which months and pathways they correspond to in Views 1 and 2.

Grouped Mode

The same dots rearrange into categorical columns grouped by: Monitoring Frequency, Threats Tracked, Entry Method, or Organization Size. White horizontal lines show the median value per column. This immediately reveals which defensive postures lead to better outcomes. For example, switching to Grouped by Monitoring Frequency shows that monthly-monitored organizations cluster at higher infection rates than daily-monitored ones.

Drill-Down

Click any dot to see the full incident record: organization type and size, entry method, infection rate, recovery time, data restored, facilities affected, and all binary outcomes (encrypted, stolen, backup lost, ransom paid, monitoring frequency, threats tracked).

Centroid Markers

When dots are brush-selected, two diamond markers appear: a gray diamond at the center of all visible data (baseline), and a green diamond at the center of the selected subset. The distance between them shows how different your selection is from the overall population.
